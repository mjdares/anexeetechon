# SEO Blog Post Template

A comprehensive template for creating blog posts optimized for traditional search engines (Google, Bing), generative AI engines, and conversion.

---

## 1. Input Details (Complete Before Writing)

| Field | Description | Example |
|-------|-------------|---------|
| **Target Audience** | Industry, role, experience level | Manufacturing engineers, operations managers, mid-senior level |
| **Primary Keyword** | Exact phrase to rank for | industrial IoT platform |
| **Secondary Keywords** | 5-15 related terms, brands, features | SCADA, HMI, OPC UA, MQTT, edge computing, Industry 4.0 |
| **Search Intent** | informational / commercial / transactional / navigational | commercial |
| **Brand Voice** | formal / conversational / bold / educational | educational |
| **Target Word Count** | Pillar: 1,800-2,200 / Standard: 800-1,200 | 1,800-2,200 words |
| **Goal** | rank, educate, generate leads, product signups | generate leads |

---

## 2. Overall Content Requirements

### Answer-First Approach
- Start each major section by directly answering the implied question in 2-3 sentences
- Add context and examples after the direct answer
- Each paragraph should be "answer-ready" for AI extraction

### Structure Guidelines
- **One idea per section**: Each H2/H3 covers a single clear concept
- **Scannability**: Short paragraphs (1-3 lines), bullet lists, tables
- **Match intent**: Content format matches what users expect for this keyword
- **Original value**: Include unique data, frameworks, or step-by-step processes

---

## 3. Title, URL & Metadata

### Title Tag (SEO Title)
- **Length**: Under 60 characters
- **Structure**: Primary keyword near beginning + clear benefit/outcome
- **Examples**:
  - `[Primary Keyword]: [Specific Benefit] in [Timeframe/Context]`
  - `X Ways to [Desired Outcome] with [Primary Keyword]`

### H1 (On-Page Main Heading)
- Exactly ONE H1 per page
- Can be slightly longer than title tag
- Include primary keyword naturally

### URL Slug
- Short, descriptive
- Include main keyword
- Avoid stop words (the, a, and, etc.)

### Meta Description
- 140-160 characters
- Include primary keyword once
- Clear benefit + soft CTA

---

## 4. Heading Hierarchy

```
H1 - Page Title (exactly one)
  H2 - Main Section 1
    H3 - Sub-section 1.1
    H3 - Sub-section 1.2
      H4 - Detail/sub-step (optional)
  H2 - Main Section 2
    H3 - Sub-section 2.1
  ...
```

### Recommended H2 Topics (Map to User Questions)
- "What is [primary keyword]?"
- "How does [primary keyword] work?"
- "Benefits of [primary keyword] for [audience]"
- "Step-by-step: How to [task] with [primary keyword]"
- "Common mistakes with [topic] and how to fix them"
- "FAQs about [primary keyword]"

## 5. Typography & Visual Guidelines

| Element | Desktop Size | Notes |
|---------|--------------|-------|
| Body text | 16-18px | Line-height: 1.5-1.8 |
| H1 | 28-36px | Bold |
| H2 | 22-28px | Bold |
| H3 | 18-22px | Semi-bold |
| H4 | 16-18px | Semi-bold |
| Paragraph width | ~60-80 characters | Max content width: 640-720px |

**Formatting Rules**:
- Use **bold** only for key phrases, stats, and action steps
- Sans-serif font (Inter, Roboto, Open Sans)
- High contrast (#111 or #222 on white)

## 6. Section-by-Section Layout

### a) Introduction (80-150 words)
```markdown
[Hook with pain point, question, or bold statement tied to primary keyword]

[What reader will learn + who it's for]

[Primary keyword in first 1-2 sentences]

[Clear promise of value to encourage scrolling]
```

### b) Definition / Core Concept (H2)
```markdown
## What is [Primary Keyword]?

[Concise, plain-language definition in 2-3 sentences]

[Why it matters + connection to reader's business/role]

| Feature | Option A | Option B |
|---------|----------|----------|
| Use case | ... | ... |
| Pros | ... | ... |
| Cons | ... | ... |
```

### c) How It Works / Key Components (H2)
```markdown
## How [Primary Keyword] Works

[Brief overview in 2-3 sentences]

1. **Component 1**: [Most important words first] - description
2. **Component 2**: [Most important words first] - description
3. **Component 3**: [Most important words first] - description
...
```

### d) Benefits & Use Cases (H2)
```markdown
## Benefits of [Primary Keyword] for [Audience]

- **[Benefit 1]**: [Mechanism] + [metric or example]
- **[Benefit 2]**: [Mechanism] + [metric or example]
- **[Benefit 3]**: [Mechanism] + [metric or example]
...
```

### e) Step-by-Step Implementation (H2)
```markdown
## How to Implement [Primary Keyword]: A Step-by-Step Guide

### Step 1: [Action]
[One-sentence bottom line]
[2-4 sentences of supporting detail]
[Tool recommendation if applicable]

### Step 2: [Action]
...
```

### f) FAQs (H2)
```markdown
## Frequently Asked Questions About [Primary Keyword]

### [Natural language question likely asked in AI search]?
[Direct answer in 1-2 sentences]
[Optional additional context]

### [Another natural language question]?
...
```

### g) Common Mistakes / Myths (H2)
```markdown
## Common [Topic] Mistakes and How to Avoid Them

1. **Mistake**: [Description]
   - **Why it hurts**: [Impact]
   - **What to do instead**: [Solution]

2. **Mistake**: [Description]
   ...
```

### h) Tools / Checklist (H2)
```markdown
## [Topic] Implementation Checklist

- [ ] [Actionable item 1]
- [ ] [Actionable item 2]
- [ ] [Actionable item 3]
...

**Recommended Tools**:
- [Tool 1]: [What it helps with]
- [Tool 2]: [What it helps with]
```

### i) Conclusion + CTA (H2)
```markdown
## Key Takeaways

- [Takeaway 1]
- [Takeaway 2]
- [Takeaway 3]

[2-3 sentence summary of key outcome]

[Single, specific CTA aligned with business goals]
```

## 7. On-Page SEO Requirements

### Keyword Placement
- [ ] Title tag
- [ ] H1
- [ ] First 100 words
- [ ] At least one H2
- [ ] Conclusion
- [ ] Image alt text

### Internal Links
- 2-6 relevant internal pages
- Descriptive anchor text (not "click here")

### External Links
- 1-3 authoritative sources
- Support claims or definitions

### Table of Contents
- Include near top of article
- Link to H2 anchors

---

## 8. GEO / AI-Search Optimization

### Answer-Ready Paragraphs
- Each 2-4 sentence block should fully answer a sub-question
- No dependency on surrounding content for comprehension

### Clear Headings
- Mirror how users ask questions naturally
- Use explicit, descriptive language

### Entity Clarity
- Avoid vague pronouns
- Repeat entity names instead of "it" or "this"
- Example: "industrial IoT platform" instead of "it"

### Formatting for AI Extraction
- Clean, minimal formatting
- Highlight definitions, steps, checklists
- Mention years where relevant (e.g., "in 2025")

---

## 9. Quality Checklist

Before publishing, verify:

- [ ] No fluff or filler - every section answers a question or supports a decision
- [ ] Active voice with concrete verbs and specific examples
- [ ] Sentences under 20-25 words (mostly)
- [ ] Paragraphs 1-3 sentences
- [ ] Keywords appear naturally, not stuffed
- [ ] Key Takeaways section with 3-5 actionable bullets
- [ ] Single clear CTA at the end
- [ ] All images have descriptive alt text
- [ ] Internal and external links included
- [ ] Meta description is 140-160 characters
